**2. Transient state filter:**
```rust
const EDGE_THRESHOLD_U64: u64 = 940_000;     // Upper bound ($0.94)
const MIN_VALID_COMBINED_U64: u64 = 900_000; // Lower bound ($0.90)

// Combined must be in realistic arbitrage window
if combined <= EDGE_THRESHOLD_U64 && combined >= MIN_VALID_COMBINED_U64 {
    // Valid edge detected
}
```

### Why False Edges at $0.02?

Polymarket batches YES and NO updates in the same WebSocket `price_changes` array. When market swings (e.g., YES drops $0.99→$0.01):

1. memchr processes YES update first → orderbook: YES=$0.01
2. Complement NO still has OLD stale value → orderbook: NO=$0.01
3. Combined = $0.01 + $0.01 = $0.02 (impossible for valid CTF)

**Fix:** Lower bound filter rejects transient states without two-pass parsing (saves microseconds).

pub fn init_signer(private_key_hex: &str) -> Result<PrivateKeySigner, String> {
    let pk = private_key_hex.trim_start_matches("0x");
    PrivateKeySigner::from_str(pk).map_err(|e| e.to_string())

pub fn generate_salt() -> alloy_primitives::U256 {
