## Session: Ghost Detection Depth Fix (04:19-04:57 EDT)

### Critical Bug: NO Depth Always Zero
**Symptom:** Ghost detection showing 0 ghosts despite ARBs being detected
**Root Cause:** Polymarket WebSocket does NOT send `outcome` field in `price_changes` messages

**Raw WebSocket message:**
```json
{
  "market": "0x0c7dd5...",
  "price_changes": [
    {"asset_id": "9352736834...", "price": "0.19", "size": "66.4", "side": "BUY"},
    {"asset_id": "3637350689...", "price": "0.81", "size": "66.4", "side": "SELL"}
  ]
}
```

**Buggy code defaulted to "yes":**
```rust
let outcome = change["outcome"].as_str()
    .unwrap_or_else(|| "yes".to_string());  // ← ALWAYS YES!
```

**Result:** All updates marked as YES, so `no_depth` was ALWAYS 0

### The Fix
Use `token_side` mapping to determine YES vs NO from `asset_id`:

```rust
let (yes_depth, no_depth) = {
    let mapping = state.token_side.lock();
    match mapping.get(&update.token_id) {
        Some((_, is_yes)) => {
            if *is_yes { (update.size, 0.0) }
            else { (0.0, update.size) }
        }
        None => (0.0, 0.0)
    }
};
```

- Formula: `depth < start_depth * 0.5` → flagged any depth drop
- But our own fills naturally consume liquidity!
- 40% ghost rate was actually **false positives**

### Fix Applied
**New formula:**
```rust
True_Ghost = Initial_Depth - Final_Depth - Our_Filled_Shares
```
