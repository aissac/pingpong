**File**: `GabagoolDirectionalEngine.java` (modified)

**Signal Logic:**
- Price 0.70+ → SELL (fade overpriced) at 0.3x size
- Price 0.55-0.70 → HOLD (no edge)

- Price <0.30 → SELL at 0.3x size (cap downside)

**Asymmetric Payoff:**
- Wrong direction: lose $3-7 (capped via `directionalMaxLossPerSlot=10`)
- Right direction: win $15-25 (2-3x size)

## Root Cause Investigation (14:30-14:51 ET / 18:30-18:51 UTC)

### Problem
Executor WebSocket goes stale (`tobFresh=0`) every ~5-10 minutes, causing trading to stop. Strategy can't reach executor HTTP API when this happens.

### Root Cause Identified
- Executor Market WebSocket stops receiving fresh data
- `tobFresh` drops from 4 → 2 → 0 over ~2-3 minutes
- Internal reconnection logic (`ClobMarketWebSocketClient`) has 60s stale timeout but doesn't properly reconnect
- After `tobFresh=0`, no new trades occur

### Pattern Observed in Logs
```
14:20:02 - tobFresh=4 ✅ (healthy)
14:21:47 - tobFresh=2 ⚠️ (degrading)
14:22:17 - tobFresh=0 ❌ (stale - no fresh data)
```

### Solution Deployed
Created **TobFresh Watchdog** (`/home/aissac/polybot/scripts/executor_watchdog.py`):
- Monitors `tobFresh` value every 20 seconds
- Restarts executor early when `tobFresh <= 2` for 3 consecutive checks
- Rate limited to 12 restarts/hour with 3-minute cooldown
- PID: 3125195 (running)
