## Session 2026-04-09 ~16:13 ET: MemPalace MCP Wiring

### Problem
MemPalace (v3.0.0, 178 drawers, ChromaDB-based) was installed and working via CLI but **never wired into OpenClaw**. Only `jcodemunch` MCP server was connected. MemPalace's semantic search, knowledge graph, and diary tools were inaccessible to OpenClaw agents.

### Root Cause
- MemPalace was installed as a Python package (`pip install mempalace`)
- It has a working MCP server: `python3 -m mempalace.mcp_server` (stdio transport)
- But it was never registered in mcporter config
- OpenClaw's built-in `memory_search` has a SEPARATE bug: calls `/api/embeddings` (old Ollama API) but Ollama v0.18.3 only supports `/api/embed` — this is unrelated to MemPalace

### Fix Applied
- Ran `mcporter config add mempalace --transport stdio --command "python3 -m mempalace.mcp_server"`
- Verified: 19 tools registered, server healthy
- Key tools now available: `mempalace_search`, `mempalace_status`, `mempalace_kg_query`, `mempalace_kg_add/invalidate`, `mempalace_diary_write/read`, `mempalace_add_drawer/delete_drawer`, taxonomy, tunnels, graph stats, duplicate check

### Open Issue

- DEE confirmed this is NOT an update issue — it's a wiring/config problem
